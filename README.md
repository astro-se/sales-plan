# sales-plan

EC事業向けの需要予測ベース販売計画CSV生成アプリです。  
Google Sheets/CSVをデータベースとして、6カ月先の必要数量を算出し、基幹取込用のCSV形式に整形します。

## 目的

- 6カ月後に「何が何点必要か」を出す
- 根拠が過去実績に基づくかを判定する
- 根拠が弱いSKUを人為判断対象として抽出する
- 最終出力CSVを固定フォーマットで生成する

## 起動

```bash
pip install -r requirements.txt
streamlit run app.py
```

## デプロイ

Streamlit Community Cloudでは、GitHubリポジトリ、ブランチ、エントリファイルを指定してデプロイします。エントリファイルは `app.py` です。Secretsはリポジトリにコミットせず、Streamlit側のSecrets管理に設定してください。

## 入力CSV

### product_master.csv

|列|内容|
|---|---|
|商品コード|SKU|
|カテゴリ|出力カテゴリ|
|単価|見積単価|
|得意先コード|例: 0013|
|納入期限コード|例: 75:楽天6カ月|
|担当コード|例: 34|
|倉庫コード|例: 18|
|色|任意|
|サイズ|任意|
|安全在庫日数|例: 14|
|発注ロット|丸め単位|
|最小発注数|MOQ|
|廃番フラグ|TRUEで除外|

### sales_history.csv

|列|内容|
|---|---|
|日付|売上日|
|商品コード|SKU|
|数量|販売数量|
|売上金額|売上金額|
|チャネル|楽天/Yahoo/Amazonなど|

### inventory.csv

|列|内容|
|---|---|
|商品コード|SKU|
|現在庫|現在庫|
|入荷予定|入荷予定|
|引当済|引当済|

### overrides.csv

|列|内容|
|---|---|
|商品コード|SKU|
|上書き数量|手動数量。空欄なら自動計算|
|除外フラグ|TRUEで出力除外|
|理由|備考に反映|

## 計算思想

1. SKU別に月次販売数を作る
2. 直近3カ月、6カ月、12カ月をブレンドして月次基準需要を作る
3. カテゴリ季節性を乗せる
4. 直近12カ月の傾きからトレンド係数を作る
5. 6カ月需要を算出する
6. 有効在庫 = 現在庫 + 入荷予定 - 引当済
7. 必要数量 = 6カ月需要 + 安全在庫 - 有効在庫
8. MOQ/ロットで丸める
9. 履歴量に応じて根拠レベルを付与する
10. 根拠が弱いSKUを要確認として抽出する

## 出力CSV列

カテゴリ, 見積日, 得意先コード, 件名, 納入期限コード, 担当コード, 摘要, 商品コード, 倉庫コード, 色, サイズ, 数量, 単価, 金額, 備考
